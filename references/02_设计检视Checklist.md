# 设计检视 Checklist (v2.0 全面增强版)

> [!IMPORTANT]
> **使用说明与分类逻辑：**
> - **公共子维度（子维度 1-4）**：通用基本体验、通用设计原则与文案规范，走查任何界面时均为**默认必检项**。
> - **特定场景/功能子维度（子维度 5-25）**：特定功能类型规范，在【检视维度二：Checklist 检视】阶段，AI 应**根据被检界面/方案的具体功能特征**，对照下方“快速匹配指南与索引表”自动匹配并启用对应的子维度进行逐条核查。
> - **GPT / AI 体验特定子维度（子维度 26）**：针对嵌入 Safety GPT、AI 助手、智能驾驶/辅助驾驶及生成式对话等 AI 场景的专项走查维度。
> - **性能与响应 SLA 附录**：提供 Web 控制台及 GPT 响应时延的定量体验验收基线。

## 一、 公共子维度（通用基本体验与设计原则，默认必检）

## 公共子维度 1: 视觉与规范一致性 (Visual & Spec Consistency)
- [ ] **色彩系统**: 是否严格遵循设计规范色板？告警红、安全绿、高危橙等语义色是否全站一致且严禁混用？
- [ ] **字体排版**: 页面标题、模块标题、正文、辅助文字的字号、字重及颜色等级是否清晰有序？
- [ ] **图标语义**: 图标风格是否统一，图标语义是否清晰准确（避免隐喻混乱或无对照说明）？
- [ ] **间距与栅格**: 栅格系统是否对齐，模块间距与组件边距是否严格遵循 8px/4px 倍数原则？
- [ ] **跨系统视觉一致**: 相似模块、跨应用/跨系统的视觉风格、图形传达及布局逻辑是否保持一致？

## 公共子维度 2: 交互效率与简单易用 (Efficiency & Ease of Use)
- [ ] **信息密度与留白**: 列表与详情页信息排布是否合理？页面清爽无视觉压迫，图文比例合适，无大片密密麻麻的文本？
- [ ] **操作入口与层级**: 核心操作入口（导航菜单、主按钮等）是否显眼易找？主次操作收纳是否得当？
- [ ] **极简操作流程**: 配置/操作任务原则上控制在 3 步以内，复杂任务最多不超过 5 步？
- [ ] **表单控制项**: 表单必填项是否控制在 8 项以内？非核心/高级配置是否默认收起并提供合理的默认值？
- [ ] **交互流连续性**: 核心任务是否在当前页/弹窗/抽屉完成，做到“非必要不跳转”？若跳转后返回，能否记忆之前的筛选项、排序及字段设置？
- [ ] **高效快捷操作**: 针对大数据量或高频运维场景，是否提供批量处理、一键套用或快捷引用能力？

## 公共子维度 3: 认知、反馈与防错机制 (Cognition, Feedback & Fault Tolerance)
- [ ] **概念表达与引导**: 名词概念是否简单易理解无歧义？信息布局与引导是否让用户一眼看懂“这是做什么的”？
- [ ] **即时状态反馈**: 所有的交互操作是否均在 2 秒内给予明确反馈（包括 Loading、Success、Toast、Empty 无数据提示）？
- [ ] **提前告知与自动纠错**: 输入框仅支持数字时是否禁止非法字符输入？空格/前后空格是否能自动清洗与自动纠错？
- [ ] **高危防错与二次确认**: 危险或不可逆操作（如删除规则、中断任务、批量解封）是否有模态强提示二次确认？必要时视情况增加动态令牌或防误触校验？
- [ ] **建设性错误引导**: 报错提示是否避免技术代码输出，且明确告知错误原因并给出清晰可落地的解决方案？

## 公共子维度 4: 文案与专业表述审查 (Copywriting Review)
- [ ] **专业术语统一性**: 界面表达是否准确专业？专业术语（如：虚拟机、镜像、负载、链路等）全文统一，严禁混用变体（如：虚机、映像、拓补图）或使用低级错别字（如：登陆/帐号/其它/秘钥）？
- [ ] **状态与动词规范**: 操作入口统一使用动词原形（如：创建、导出）；进行中统一使用“正在+动词”（如：正在生成）；已完成/提示语使用“已+动词”（如：已保存）。
- [ ] **禁止文案拼接**: 动态占位符是否仅填充具体实体名称？严禁出现不合规的拼接句式（如：`{action} 失败`）。
- [ ] **模板句式标准化**: 常见交互场景是否严格套用模板句式：
  - 二次确认：`确定要{操作}{对象}吗？`
  - 未选择对象：`请先选择{对象}`
  - 按钮置灰：`{原因}，不支持该操作。`
  - 占位/引导：`请选择{对象}` / `请输入{对象}`
- [ ] **数值与单位格式**: 数字与物理/时间/计数单位之间是否有空格（如：`10 MB`、`5 分钟`、`5 K`）？计数单位是否严禁使用“万、亿、W”？
- [ ] **引用与标点格式**: UI 路径引用统一使用【】加 `>`（如：`【系统管理】>【服务管理】`）？标点符号符合中文全角/英文半角规范，无冗余非白名单特殊字符？
- [ ] **双语与合规适配**: 海外版/多语言界面空间预留是否充足（避免英文溢出或折行排版错乱）？表达是否符合当地法律法规与文化习惯，无歧义或冒犯？

## 二、 特定场景/功能子维度（条件匹配启用）

## 特定子维度 5: 总览仪表盘类 (Dashboard)
- [ ] **价值定位**: 是否聚焦目标用户关键 Job (如：工程师每日运维、向上汇报)？信息呈现是否围绕叙事逻辑？
- [ ] **界面布局**: 是否采用卡片式布局？逻辑是否遵循使用场景？是否支持自定义卡片顺序/显隐？
- [ ] **信息简洁**: 层次分明，分区清晰，重点突出？是否尽量可视化呈现，避免了大篇幅文字或表格？
- [ ] **点击下钻**: 关键数据（待办、风险等）是否支持直接点击下钻查看详情？
- [ ] **数据逻辑**: 取值范围是否清晰？是否存在逻辑错误（如：子项之和不等于总数）？下钻数据是否能对齐？
- [ ] **数据时效**: 实时性数据（如拦截数）更新是否及时？(原则上实时，最大延迟不超过 2 分钟)。
- [ ] **无数据场景**: 是否有无数据场景的价值传递或情感化设计？(不可不做任何处理)。
- [ ] **加载性能**: 默认加载时长是否达标？(SaaS 产品 < 3s，硬件/盒子类 < 5s)。

## 特定子维度 6: 大屏展示类 (Large Screen Display)
- [ ] **价值定位**: 是否聚焦监控和汇报场景（如日常运维大盘、领导检查等），信息呈现围绕目标用户关键 Job 叙事？
- [ ] **布局与主图**: 板块布局是否遵循叙事逻辑，主图设计逻辑是否符合大屏核心价值，清晰易懂？
- [ ] **信息呈现**: 层次分明，分区清晰，重点突出，完全可视化呈现，严禁大篇幅文字及复杂表格？
- [ ] **板块自说话**: 每个板块传递的信息是否简单易懂、一目了然，无需过多人工解读？
- [ ] **少交互与自动展开**: 大屏尽量减少手动交互（仅保留简单下钻），折叠信息是否能按规律自动轮播展开/隐藏？
- [ ] **数据逻辑与对齐**: 默认数据取值范围清晰，无子维度相加不等于总数或大屏与下钻数据不一致的问题？
- [ ] **数据更新时效**: 实时攻击/拦截数等时效数据更新及时，最大延迟不超过 2 分钟？
- [ ] **视觉效果与动效**: 主图视觉符合大屏定位，具备流畅炫酷的动画过渡效果，提升视觉冲击力？
- [ ] **加载性能**: 大屏默认加载时长在 3 秒以内，最长不得超过 5 秒？

## 特定子维度 7: 运营报告类 (Operation Report)
- [ ] **价值定位**: 模版场景定位是否清晰 (运营类/汇报类)？结构是否围绕场景叙事？
- [ ] **结构设计**: 是否遵循“总-分-总”或“总-分”结构？关键信息是否在总结模块清晰呈现？
- [ ] **章节逻辑**: 信息呈现层次分明，章节逻辑清晰，重点突出？
- [ ] **色彩搭配**: 是否专业、稳重，适合长时间阅读，避免过于花哨？
- [ ] **导出灵活性**: 是否考虑了用户二次编辑场景？(如提供 Word 格式)。
- [ ] **交互合理性**: 离线文档（PDF/Word/PPT）内是否存在信息折叠或按钮交互？(除 HTML 报告外，不允许存在需要交互的操作，但支持内部跳转)。
- [ ] **数据逻辑**: 数据间逻辑是否清晰，无明显歧义或计算错误？
- [ ] **文案质量**: 措辞得当，简洁有重点？是否存在语病或错别字？
- [ ] **无数据场景**: 报告中无数据模块是否有妥善的处理或解释？

## 特定子维度 8: 配置管理类 (Configuration Management)
- [ ] **流程极简**: 一个配置任务原则上操作 <= 3 步，复杂任务 <= 5 步？是否在当前页/抽屉完成，避免页面跳出？
- [ ] **配置项清晰**: 必填项是否 <= 8 项？非必填是否提供默认值或放入高级配置？必填项是否红星标注？
- [ ] **引导提示**: 是否结合小 i、占位符进行提示？关键配置旁是否提供案例参考（如高危端口使用）？
- [ ] **操作反馈**: 反馈是否及时（加载动画、明确的成功/失败弹窗/消息提示）？
- [ ] **保存与恢复**: 复杂配置（>8项）是否提供自动保存机制？崩溃或异常丢失后能否恢复编辑内容？
- [ ] **中断恢复**: 步骤 >= 3 步的长流程是否支持随时中断并保存进度，后续继续编辑？
- [ ] **操作便捷性**: 是否提供批量操作、默认模板、或快速定位策略等交互？
- [ ] **风险二次确认**: 高危或不可逆操作（如规则删除、大规模关闭终端）是否进行二次确认？必要时是否有动态令牌验证？
- [ ] **审计记录**: 是否支持历史操作记录管理，方便回溯（谁在什么时间操作了哪个字段）？

## 特定子维度 9: 部署类 (Deployment)
- [ ] **引导流程**: 复杂任务是否分步引导？核心 <= 3 步，复杂 <= 5 步？开始前是否有步骤预览？
- [ ] **进度反馈**: 耗时较长的部署是否提供阶段性进展展示？是否提供进度条？
- [ ] **配置项简洁**: 必填项控制在 8 项以内？是否有清晰的名称描述与红星标注？是否提供案例参考？
- [ ] **开箱即用**: 针对不同诉求客户，是否提供内置的开箱即用模板（SaaS 场景）或预设规则？
- [ ] **防错指引**: 是否实时校验格式错误？错误提示是否包含明确的修复指引（如：告知具体的数值要求）？
- [ ] **故障自闭环**: 报错信息是否具备可读性，尽可能指导一线人员自行闭环解决故障？
- [ ] **极端场景健壮性**: 长表单是否提供数据暂存？断网提交是否有友好提示？强制崩溃恢复后数据是否留存？

## 特定子维度 10: 终端相关类 (Client / Endpoint)
- [ ] **客户端运维管理**: 支持版本集中管控与静默升级？离线状态下具备基本自保与日志缓存能力？系统 CPU/内存资源占用是否有严格上限控制？
- [ ] **客户端通用体验**: 运行状态清爽无侵入，任务通知轻量？常驻托盘/图标语义明确，一键打开主界面？支持个性化资源占用调优？
- [ ] **客户端部署交互**: 安装包体积轻量，提供一键极简安装？安装过程具备静默环境预检与自动注册关联平台能力？

## 特定子维度 11: 运营管理类 (Alerts/Assets/Events)
- [ ] **布局直观**: 筛选、统计、列表、操作区布局是否简洁？用户能否在 2s 内理解分区？屏占比是否合理？
- [ ] **信息高效**: 关键信息是否能在一屏内展示完整？用户能否快速获取核心指标？
- [ ] **减法交互**: 是否做到“非必要不跳转”？任务处理（新增、详情）是否优先使用弹窗或抽屉？
- [ ] **统计联动**: 数据统计图表是否支持点击后快速过滤列表数据？
- [ ] **筛选能力**: 重筛选场景下，表头列是否支持快速筛选？是否提供简易与复合两种筛选模式？
- [ ] **操作批量**: 是否提供批量修改状态等操作以减少冗余步骤？
- [ ] **状态记忆**: 返回后是否能记住之前的筛选条件、排序及自定义显示的字段？

## 三、 Safety GPT / AI 体验质量与交互标准

## 特定 AI 子维度 26: Safety GPT / AI 体验与交互类 (GPT & AI)

### 26.1 对话互动与唤起
- [ ] **唤起标识清晰**: AI 交互入口（如智能驾驶、侧边栏助手、卡片浮图标）视觉标识统一且不强行遮挡核心业务区。
- [ ] **主动智能推荐**: GPT 根据用户当前操作上下文或异常告警，主动推荐下一步处置意图并给出简要推荐依据。
- [ ] **能力边界告知**: 在交互初始或引导页，客观准确告知 AI 的适用范围与能力局限，降低误用期待。

### 26.2 意图表达与共识
- [ ] **高效意图引导**: 支持根据输入语境自动补全、追问推荐（Suggested Follow-ups），并提供指令库/常用 Prompt 一键调用。
- [ ] **授权下执行 (Human-in-the-Loop)**: 涉及更改规则、下发处置等高危操作时，AI 必须经由用户显式确认授权后方可执行，严禁越权自动打洞/封锁。
- [ ] **自然流畅交流**: 语言表达接近人类专业安全专家风格，不刻板生硬，并支持对话上下文纠正与补充。

### 26.3 结果展示与思维透明化
- [ ] **思维过程可视 (CoT)**: AI 的信息加工处理过程、推理步骤及分析逻辑（Thought Stream）清晰可视，支持展开/收起。
- [ ] **举证有据可依**: AI 输出的结论（如告警判定、攻击者画像）必须附带原始日志、实体上下文或规则来源等可视化证据链（Evidence Links）。
- [ ] **即时与阶段反馈**: 对 AI 生成响应给予打字机流式（Streaming）即时反馈；长任务提供预计等待时间与后台异步生成能力。

### 26.4 失败容错与优化反馈
- [ ] **优雅处理失败**: 当 AI 无法理解意图时，明确提示原因并提供快捷修正示例、推荐相关搜索或一键转人工/工单通道。
- [ ] **结果评价与持续学习**: 提供赞/踩（Thumbs up/down）及修改反馈入口，反馈后展示调整后的优化效果，强化成长感知。

### 26.5 GPT 性能体验 SLA
- [ ] **首字响应时延 (TTFT)**: 简单问答/告警分析首字响应时延预期 <= 3 秒，底线 <= 5 秒。
- [ ] **生成完成总时延**: 完整分析报告或复杂事件解读首字到完成时延预期 <= 20 秒，底线 <= 30 秒。

## 附录：性能与响应时间体验基线 (Performance & Response SLA Baseline)

在标准网络环境与 80% 常见业务负载下，Web 控制台及核心页面必须达到以下性能体验指标：

| 体验维度 | 指标类型 | 预期合格标准 (P75) | 极限底线标准 | 适用场景 / 备注 |
| :--- | :--- | :--- | :--- | :--- |
| **页面整体加载** | 完整可见渲染 | **<= 3.0 秒** | **<= 5.0 秒** | SaaS 类产品 < 3s，盒子/硬件控制台 < 5s |
| **大屏 / 看板** | 首屏图表完成 | **<= 3.0 秒** | **<= 5.0 秒** | 态势感知大屏、综合运维看板 |
| **前端白屏 (FCP)** | 首次内容绘制 | **<= 1.8 秒** | **<= 2.5 秒** | 全站所有 Web 页面 |
| **后端 API 响应** | RT (Response Time) | **<= 1.0 秒** | **<= 2.0 秒** | 普通查询与提交 API (大文件导出类例外) |
| **拓扑图渲染** | 100节点画布加载 | **<= 3.0 秒** | **<= 4.0 秒** | 拓扑图、数据流转图 |
| **GPT 首字响应** | TTFT (First Token) | **<= 3.0 秒** | **<= 5.0 秒** | Safety GPT 对话与智能分析 |
| **GPT 全文生成** | Complete Generation| **<= 20.0 秒** | **<= 30.0 秒** | 复杂事件 AI 归因解读与处置生成 |
